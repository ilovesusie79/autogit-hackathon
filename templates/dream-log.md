A quiet place to write down a dream the moment after waking. One page, one column, the type of a book of letters rather than the type of a product. The app opens to a blank entry for today, the cursor already inside the body, the date set, nothing in the way. You write, you tag, you close the tab, and the entry waits for you in a list that grows over the years.

The page sits on a deep ink color, almost violet, the color of the inside of an old eye closed against morning light. The text is a soft lavender that does not glow. There is a single warmer color, a faint gold, used only for the date of an entry that contained the lucid tag and for the small chevrons that mark navigation. Type is Lora throughout the page, loaded from Google Fonts, weights 400 and 500 and italic 400. Body text is 18px with a generous line height of 1.7. Headings inside entries, if you write any, render in 22px. Everything else is the same size as the body. Nothing is bold by default. There are no icons except a small inline svg moon in the corner that does not click anything.

When the page loads, you see a column at most 640px wide centered on the page. At the top in 14px italic muted lavender is the word dreams, lowercase, nothing more. Below it on a new line is the date of the current entry in long form like Friday the twenty ninth of May. To the right of the date in the same line is a small chevron, gold, that opens an older entry. To the left another chevron that goes back to today. The active entry sits below this line as a single tall multi line text field with no border and no background, exactly the same color as the page, the only thing that gives it shape is the cursor inside it. Below the entry, a soft row of small tag pills, each rendered as the tag word in 13px italic with a thin underline. The tag row is editable inline, you type a word followed by a comma to add it, you click an existing pill to remove it. Below the tags is a single line in 12px dim lavender that shows the time the entry was last edited in a human form like saved one minute ago.

The list of older entries does not live on the same page as the current entry. To see them, you click a small word at the bottom of the page that reads earlier entries, lowercase, italic. The current entry slides up out of view and the list takes its place. Each entry in the list is two lines, the date on the first line in 16px regular, and a single line preview of the body in 14px italic muted lavender. Entries that were tagged lucid have their date rendered in the warm gold. Entries that were tagged nightmare have a small faint outline circle to the left of the date, no fill, no symbol inside. There are no other tag markers on the list, the rest of the tags appear only on the entry page. Tapping or clicking a row opens that entry as the active one. Inside an older entry the navigation chevrons walk forward and back through the history one entry at a time. A button at the top right reads today, gold, italic, and returns you immediately to the entry for today. If there is no entry for today yet, that button creates one.

Tags are not preset, they are whatever you type. The app keeps an internal frequency table of tags and offers a small autosuggest under the tag input when you type two or more letters that match the start of an existing tag in your history. Suggestions appear as a stack of up to five small lavender pills, you can pick one with the mouse or press tab to accept the top one. The two tags lucid and nightmare have small effects on the rest of the app as described above, but they are not special in any other way. You can use them, ignore them, invent your own. A small search box lives only on the list view, single line, italic placeholder reading search dreams. Typing in it filters the list to entries whose body or tags contain the query, case insensitive. The current entry page never shows search.

Everything you write lives in localStorage under the key dream.log.v1. The shape is a single object with a map of entries keyed by iso date and a flat tags table that tracks frequency. Each entry has a date, a body string, an array of tag strings, an iso created at timestamp, and an iso edited at timestamp. The list is rendered by reading the entries map and sorting keys descending. On boot, if the storage is empty, the app seeds a single example entry dated three days ago, body reading I was on a slow train moving through a town I had never seen. The walls of every house were painted the same color as the sky. I tried to ask the conductor where we were and she answered in a language I almost remembered, with tags train and slow and gold and shape of a word. The seed is there so you understand what an entry feels like, not so you understand how to use the app. If the storage already holds entries, the seed is not added. If you delete every entry the seed does not return. There is a small text button at the very bottom of the list view that reads erase everything in 12px dim lavender, with a confirm prompt that says are you sure you want to erase every dream in this log, and a small cancel option in the same prompt. There is no undo.

Writing is autosave. Every keystroke updates the in memory entry, every 500ms of idle time writes it to storage. The footer line updates to saved just now after every successful write. If localStorage is unavailable, the footer instead reads this session is not being saved, and the app keeps working in memory until the tab closes. There is no spinner anywhere. There is no skeleton. The first paint shows the current entry already drawn. There is no login. There is no cloud sync. There is no export and there is no import, because the point of a dream log is to be the kind of object you keep private. If you want to back up your entries, you do it yourself with the developer tools, the localStorage value is plain json. A small note in 11px italic muted at the very bottom of the list view confirms this in a single sentence reading your dreams live only in this browser on this device, back them up yourself if you want them to outlive a cache clear.

The build is a small Vite project. Tailwind is in use only for layout and color, all type rules are applied directly through Tailwind utility classes that point to the custom palette extended in tailwind config. The palette adds ink at the value 16142A, lavender at C9BFE4, lavender muted at 8F86A8, lavender dim at 5E5872, and gold at D9B26B. The body sets the page color to ink and the default text color to lavender. The single moon glyph in the corner is a 16px svg drawn inline in App.tsx with two paths, the outer disc in lavender at 30 percent opacity and the shadow that turns it into a crescent in ink. Files are index.html with the Lora link in the head, src/main.tsx as the entry, src/App.tsx as the single default export holding all top level state, src/components/Entry.tsx for the current entry view, src/components/EntryList.tsx for the list view, src/components/TagInput.tsx for the tag pill row with autosuggest, src/lib/storage.ts wrapping localStorage in a try catch, src/lib/dates.ts for the long form date and the human time helper, src/lib/seed.ts for the example entry, tailwind.config.ts for the palette extension, package.json with react and react dom and vite and tailwind and only those, README.md with two short paragraphs covering what the app is and how to run it. Stack is React 18 with TypeScript, Tailwind CSS, Vite. State is plain useState and useReducer. No router, no context provider, no global store, no date library, no rich text editor, no markdown parser, no external ui kit, no icon pack. Every shape on the page including the chevrons and the moon is inline svg in the component that renders it. The default export from src/App.tsx is the only entry point.

```json
{
  "entries": {
    "2026-05-26": {
      "date": "2026-05-26",
      "body": "I was on a slow train moving through a town I had never seen...",
      "tags": ["train", "slow", "gold", "shape of a word"],
      "createdAt": "2026-05-26T07:14:00.000Z",
      "editedAt": "2026-05-26T07:14:00.000Z"
    }
  },
  "tagFreq": {
    "train": 1,
    "slow": 1,
    "gold": 1,
    "shape of a word": 1
  }
}
```

The app is done when a first time visitor opens the page and sees a date, a blinking cursor inside an empty entry, and a moon in the corner. They type a line, the footer reads saved just now within a second. They add a tag by typing the word lucid and a comma. The date at the top turns gold. They click earlier entries, the list opens and shows the seeded train dream three days ago. They click it, read it, hit today, return to the blank entry. They close the tab. They open it the next morning. The current entry shows the next morning's date, ready, and yesterday's lucid entry waits in the list with its date in gold. Nothing else is in the way.
